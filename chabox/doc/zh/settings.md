# 2. 应用设置与文件管理

作为 `快速入门` 教程的延伸，本章将带您深入了解 ChaBox 的两大核心基础板块：**全局应用配置** 与 **本地文件管理器**。

---

## 2.1 全局应用配置

点击首页左上角的三横线按钮，即可滑出**左侧抽屉式菜单**。在这里，您可以一站式管理整个应用的底层行为和安全策略：

![全局应用配置](../img/home-settings.png)

### 2.1.1 密码高级管理（生命线）
密码是您数据的核心锁钥，侧边菜单提供了全方位的密码管理功能：
1. **生成新密码**：获取高强度的随机强力密码。
2. **存储密码**：在本地更新并安全记录您的当前密码，以便于日常的自动加解密操作。
3. **导入与导出**：强烈建议您使用“导出到文件”功能。将密码备份并妥善保存在外部 U 盘等物理介质中。

> [!CAUTION]
> **密码变更特别提醒**：
> 密码仅加密保存在您本地设备中。如果您决定更换新密码，请务必在更换前将旧密码加密产生的所有文件全部解密还原。否则，密码一旦变更，新密码将无法解开旧密码加密过的历史文件。

### 2.1.2 防护加固与目录配置
*   **物理防护加固**：
    *   **应用锁与自动锁屏**：设置进入门禁和闲置自动锁屏，防止您离开设备时操作现场被他人窥探。
    *   **全防泄密设计**：系统默认禁用截屏与录屏。在移动端切换应用时，会自动遮挡界面以防止安全隐私泄密。
    *   **剪贴板定向清理**：由于手机等移动端系统对频繁读取剪贴板存在限制，为了平衡安全与性能，该功能**仅在您查看/拷贝密码文本，或者编辑安全日记时**才会定向触发清理，自动抹除敏感残留。
*   **文件输出目录设置**：您可以为“打包归档”、“核心加密”、“安全日记”及“解密还原”分别配置专属的输出目录，使您的工具输出与核心隐私资产物理隔离，分类清晰互不干扰。

## 2.2 通用本地文件管理器

虽然文件管理器原本不属于配置范畴，但它是您日常管理日记和敏感资产的关键。在 ChaBox 中，**“安全日记”** 与 **“文件保险箱”** 共享同一套高效的文件管理器。

它主要由以下两大核心部分组成：

### 2.2.1 直观的文件列表

文件列表可帮助您实时监控和管理加密目录中的所有文件：

![加密箱文件列表](../img/vault-list.png)

在列表页面的右上角，我们为您准备了两个快捷按钮，帮您轻松实现文件在 ChaBox 与手机/电脑其他应用之间的“搬运”：
*   **添加文件 (Import)**：点击此按钮，您可以从手机或电脑的相册、文件夹中选取文件，直接存入当前正在查看的目录中。
*   **分享文件 (Share)**：点击此按钮，可以将当前列表里的所有文件一次性打包，发送给网盘、邮件等其他应用，方便您将文件“另存”到外部空间进行备份。

*   **支持目录树级归类（强大层级管理）**：文件列表支持**目录的递归展示**。这意味着当您在安全设置中指定了输出目录后，您依然可以在该文件夹内**自行建立任意级子目录**（子文件夹），用来更科学地给文件分类，列表均可完美穿透呈现。
*   **视觉类型归类**：列表项的**左侧**会根据不同的文件类型（如图片、文本、压缩包等）展示**不同的文件图标与代表颜色**，让您一眼即可识别文件属性。
*   **快捷操作菜单**：每个列表项的**右侧**都配备了快捷功能菜单（通常是三个小点按钮）。点击即可进行以下操作：
    *   **查看**：在内存中直接解密并预览（仅限文本类型和图像类型，且绝不在磁盘产生临时明文，确保安全）。
    *   **分享文件 (Share)**：将单个文件发送给外部应用，实现数据的快速另存。
    *   **复制**、**移动**、**删除**（包括物理覆盖级安全擦除）等。

### 2.2.2 强大的文件筛选器

当您的存储文件较多时，点击列表上方的 **“Filter (过滤)”** 按钮可呼出文件筛选器，帮助您精确定位所需数据：

![文件筛选器截图](../img/vault-filter.png)

*   **多维匹配搜索**：
    *   **路径筛选（桌面端更灵活）**：文件管理器展现出极高的通用性。在桌面端，您可以通过筛选器定位并选取任意本地文件夹，实现全盘与多存储媒介的深度检索。移动端则受到应用沙盒限制，只能访问沙盒内的目录，无法直接选取沙盒外路径。
    *   **模糊搜索（通配符匹配）**：支持使用 `Glob` 模式进行搜索，且内部开启了递归遍历。它不仅能匹配文件名，还可以匹配路径片段。
        *   `*` 只匹配单个路径段中的任意字符，通常用于当前目录层级的文件名或目录名。
        *   `**` 表示跨越任意深度目录，可用于匹配子目录中的文件。
    *   **最佳实践**：
        *   `*.txt`：匹配当前目录下所有 `.txt` 文件。
        *   `**/*.txt`：匹配当前目录及所有子目录中的 `.txt` 文件。
        *   `secret/**/*.cha`：匹配 `secret` 目录及其任意深度子目录中的 `.cha` 文件。
        *   `*日记*`：匹配文件名中包含“日记”的所有文件。
        *   建议：如果你希望跨目录查找，优先使用 `**/`；如果只需要当前目录层级，可以使用 `*`。
    *   **文件格式筛选（类型匹配）**：你可以直接按文件的格式大类筛选，比如只看“图片类型（image）”或“视频类型（video）”。系统会基于本应用的 MIME 类型识别（mimetype）自动判断这些格式，不必去记各种后缀名。
    *   **备注**：如果你想更精细地控制，打开侧边设置面板，可以加载自己扩展的文件格式。这是全局设置，会影响：
        *  `mimetype` 筛选条件的识别能力，能让更多自定义扩展名与文件格式相对应；
        *  文件列表里的类型图标和分类显示，让文件按类型呈现得更准确；
        *  自动加解密时判断“是否是文本文件”的逻辑，如果某个文件默认没识别为文本，也可以通过这里的规则让它正确处理。 
*   **二次精细过滤**：支持按照 **文件大小范围**（比如找出大于 10MB 的大文件）与 **修改日期区间**（比如找出最近一周内修改过的文件）进行二次过滤。
*   **灵活排序**：筛选出的结果可选择基于**文件大小**、**修改日期**或**文件名**进行升序或降序排列，检索和归档省时省力。

### 2.2.3 最佳实践：桌面端与移动端的使用差异

ChaBox 的“通用本地文件管理器”实际上包含两个级进功能：
1. 先选取一个目录；
2. 然后在该目录基础上进行筛选。

这两层在不同平台上的表现有所不同：

*   **移动端（Android / iOS）**：仅能选取并管理应用沙盒内的目录，沙盒外目录无法直接访问。文件管理器在移动端主要用于管理沙盒内的文件和输出目录（例如“文件输出目录设置”中的默认目录）。如果需要跨设备传输或外部备份，建议额外使用网盘功能来中转已加密的文件。
*   **桌面端（Windows / macOS / Linux）**：可通过“文件输出目录设置”指定沙盒外目录作为存储目录，直接将 ChaBox 的工作目录指向外部盘、共享盘或常用文件夹，实现本地路径上的直接管理。

另外，文件管理器目录和网盘备份是两种不同用途：
*   文件管理器目录用于将文件存放在 ChaBox 可直接管理的本地路径；
*   网盘备份则是把已加密文件复制到外部云存储作为副本。

对于已加密文件，只要密码未泄露，保留多个副本本身也是安全的。因此：
*   移动端建议：在沙盒内使用文件管理器，必要时用网盘中转已加密文件；
*   桌面端建议：既可以直接管理本地目录，也可以定期把已加密文件备份到网盘，形成“本地管理 + 外部副本”的双重保障。
