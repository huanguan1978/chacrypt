# ChaBox - 离线数字保险箱

[English](./README.md) | **简体中文**

**ChaBox** 是一款基于工业级 **ChaCha20-Poly1305** 认证加密标准构建的高性能文件安全工作站。

### 为什么在互联网时代，您仍需要 ChaBox？

在万物互联的今天，数据正以前所未有的速度在网络中穿梭。然而，穿梭意味着暴露。ChaBox 专注于保护处于以下两大高危状态的数据：

1.  **联网传输中**：即时通讯工具、云盘或电子邮件本质上都是“不安全的高速公路”。ChaBox 在数据踏上公路前完成本地硬化，确保即便是遭遇网络劫持或云端泄露，您的隐私依然稳如泰山。
2.  **离线存储中**：存放在本地磁盘、NAS 或移动媒介上的敏感资产面临物理丢失或未经授权访问的风险。ChaBox 为其建立“冷保护”，确保数据主权始终掌握在您手中。

**ChaBox 的核心哲学是：在数据踏上互联网之前，先在本地完成绝对物理隔离的加固。** 它不仅是一个加密工具，更是您在数字化生存中的“离线主权堡垒”。

## ✨ 核心优势

*   **五大原子化工具 (具有批量处理能力)**：
    1.  **批量归档** & **批量解档**：高效的结构化数据打包。
    2.  **批量加密** & **批量解密**：基于 ChaCha20-Poly1305 的工业级保护。
    3.  **数字碎纸机 (Wipe)**：物理级安全擦除，彻底销毁原始敏感文件。

*   **安全日记（核心功能：沉浸式写作 + 安全增强）**：
    *   **沉浸式 Markdown 写作与预览**：支持即写即存与实时预览，兼顾记录效率与阅读体验。
    *   **变量化引用**：支持应用变量与自定义变量占位，可统一管理图片路径与网页链接；跨平台迁移时仅需调整变量即可快速适配。
    *   **密文附图**：支持本地与 HTTP 远程密文图片嵌入，实现日记正文与附图的统一密文保护。

*   **原生文件系统主权 (像管理普通文件一样管理加密资产)**：
    不同于许多保险箱软件将文件“吞入”一个无法直接查看的大型数据库，ChaBox 坚持以**原生文件夹**组织资产，把控制权交还给您。
    *   **文件存储透明**：加密后的文件依然以清晰的目录结构存在。您可以直接通过手机或电脑自带的“文件管理器”进行移动、物理备份或云端同步，没有任何学习门槛。
    *   **文件分类自由**：文件夹本身就是最自然的“分类器”。您可以根据个人习惯建立无限层级的目录（例如：`财务/2024/发票`），让加密资产井然有序，而不是像传统数据库那样将文件堆在一个难以管理的“大池子”里。
    *   **文件移动自由**：彻底告别“数据被软件绑架”的风险。由于资产不锁死在单一数据库内，无论是更换设备还是跨工具协作，您的数据永远处于“随时可迁移”的流动状态，真正实现安全与自由的统一。

*   **内存级主动防御**：
    *   **多维锁定**：集成应用锁、闲时自动锁定。
    *   **隐私保护**：全平台防截屏、**全平台防录屏**以及切屏遮屏防窥。
    *   **剪贴板安全防护**：针对“安全日记”与“密码文本”等敏感模块，提供定时自动清理剪贴板功能，从源头阻断核心机密在系统层面的残留泄密风险。
    *   **无痕预览**：坚持“内存预览”原则，杜绝明文数据在磁盘留下任何物理碎屑。

*   **全语境安全赋能 (100% 离线)**：
    *   **多国语言矩阵**：深度适配英语、汉语（简体/繁体）、西班牙语、日语、德语、法语、葡萄牙语、韩语及俄语，确保全球用户都能零门槛掌握隐私主权。
    *   **离线安全知识库 (每日简报)**：内置多语言安全知识百科，每日轮播展示涵盖应用技巧、物理安全建议、密码学常识及安全警示录的精选信息，在完全不联网的前提下，潜移默化提升用户的安全素养。

*   **工业级算法基石**：采用 ChaCha20-Poly1305 (AEAD) 算法，兼顾极高性能与强大的完整性校验，数据一旦遭遇位翻转或恶意篡改，解密即刻失效并报警。

*   **可视化审计链**：直观的堆叠扩展名（如 `.tar.tz.cha`）清晰展示了数据的处理链路，确保过程透明且专业。

## 🛠 开发者指南 (For Developers)

本项目基于 Flutter 框架开发，遵循标准的 Dart/Flutter 编程实践。

### 技术架构规范 (Scope-Driven)
我们遵循 **“作用域驱动 (Scope-Driven)”** 的设计哲学。每个功能模块（如 `home` 或 `action`）都自包含在 `page/<scope>/` 下，自然体现 MVC 模式（C: 路由入口控制器, V: layout/widget, M: helper/utils）。

*   **架构设计指南**: 
    * [中文版](./DESIGN_zh.md) | [English](./DESIGN.md)

### AI 辅助编程规范 (AI-Assisted Coding)
我们欢迎 AI 辅助开发，但必须遵循以下契约以保障代码质量与长期的可维护性：
1. **架构契约**：AI 生成的代码必须严格遵守 ChaBox 的“作用域驱动”架构。
2. **高质量注解 (English Comments)**：所有 `helper/` 下的业务方法及各功能模块的 `Public API` 必须包含英文注释，阐述“为什么这么做”（设计意图/权衡点）。
3. **责任人机制**：AI 仅作为辅助。开发者提交代码时必须标注来源（如 `[AI Generated]` 或 `[AI Assisted]`），并确保已完成逻辑审查与功能测试。

### 贡献哲学：**为己所用，鼓励贡献**
1. **自由定制**：发现不符合您特殊工作流的地方？欢迎 Fork 并改造出最适合您的版本。
2. **鼓励回馈**：如果您实现了具有普适安全价值或效率提升的改进，欢迎提交 Pull Request。
3. **质量基准**：请确保代码通过 `flutter analyze` 检查。
4. **极简依赖原则**：严苛的依赖管理，优先使用官方标准库。
